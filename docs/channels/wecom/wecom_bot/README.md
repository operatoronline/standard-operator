# WeCom Group Bot

The WeCom Group Bot integration is the simplest way to deploy Operator. It allows the agent to exist inside a standard group chat, listen for events, and post responses via a standard webhook URL.

## Configuration

```json
{
  "channels": {
    "wecom": {
      "enabled": true,
      "token": "YOUR_TOKEN",
      "encoding_aes_key": "YOUR_ENCODING_AES_KEY",
      "webhook_url": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY",
      "webhook_path": "/webhook/wecom"
    }
  }
}
```

| Field            | Type   | Required | Description                                               |
| ---------------- | ------ | -------- | --------------------------------------------------------- |
| enabled          | bool   | Yes      | Enable WeCom Group Bot channel                            |
| token            | string | Yes      | Internal verification token                               |
| encoding_aes_key | string | Yes      | Webhook encryption key                                    |
| webhook_url      | string | Yes      | Destination URL generated by WeCom to push responses to   |
| webhook_path     | string | No       | Local path to receive webhook events from WeCom           |

## Setup Guide

1. Navigate to the **WeCom Admin Console** -> **Group Chat** -> **Add Group Bot**.
2. Configure the Bot parameters and copy the generated Webhook URL (it will look like `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx`).
3. Set up a secure callback route in your WeCom settings and note the Token and AES Key.
4. Add the credentials to your Operator configuration.
5. In group chats, the bot will only respond to messages that explicitly `@mention` it to avoid unnecessary triggering.
